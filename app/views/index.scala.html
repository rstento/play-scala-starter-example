This tutorial introduces Play Framework, describes how Play applications work, and shows you how to create a page that displays a customized Hello World greeting. If you are reading this page in your browser, congratulations! You have successfully built and run a Play application. If not, you likely opened the source index.scala.html file. Introduction to Play

As illustrated below, Play is a full-stack framework with all of the components you need to build a web application, including: an integrated HTTP server, form validation, Cross-Site Request Forgery (CSRF) protection, RESTful web services API, and more. Play integrates with many object relational mapping (ORM) layers. It supports Anorm, Ebean, Slick, and JPA out-of-the-box, but many customers use NoSQL or REST layers and other ORMs.

Play APIs are available in both Java and Scala. The Framework uses Scala and Akka under the hood. This endows Play applications with a stateless, non-blocking, event-driven architecture that provides horizontal and vertical scalability and uses resources more efficiently. See Elasticity and Efficient Resource Usage for more information.

In contrast with web frameworks that were designed to support large eco-systems, such as Java EE, Play was developed by web developers—for web development. The following describe just a few of the reasons why developers enjoy using Play framework:

  • The concise and functional programming patterns are familiar to Scala developers

  • The Model-View-Controller (MVC) architecture is a common pattern.
  • Direct support of common web development tasks and hot reloading save precious development time.
  • A large active community promotes knowledge sharing.
  • Use of Twirl templates to render pages. The Twirl template language is:
    • Easy to learn.
    • Requires no special editor.
    • Provides type safety.
    • Is compiled so that errors display in the browser.

To learn more about Play's benefits, visit the Play website. Now, let's dive into what a Play application looks like.
